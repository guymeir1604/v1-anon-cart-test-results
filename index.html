Issues Found (4) — Root Cause Analysis
DO-2.1 BUG Free privacy IS showing in domain-only flow
View Screenshot
Root Cause: Privacy pricing logic applies "free forever" to ALL anonymous flows without distinguishing domain-only vs bundle. The free privacy eligibility check doesn't filter by flow type.
Location: premium-store-core/.../PremiumStoreModelBuilder.scala — privacy pricing enrichment
Suggested Fix: Add a flow-type condition to the privacy pricing logic. When flowType == DOMAIN_ONLY, skip the free privacy override and use the standard price. The flow type is available in the cart context from the storefront referral info.
PP-1.1 BUG "Free domain for 1 year" is NOT first feature on plan cards
View Screenshot
Root Cause: Plan card features list order is controlled by the product catalog configuration. Currently "100 collaborators" is first. The "Free domain for 1 year" feature exists in the list but is not prioritized to position 1.
Location: premium-product-catalog/ — feature ordering config, or Eclipse SDK package picker FeaturesGrid component
Suggested Fix: Update the features sort order in the product catalog to place the "Free domain for 1 year" benefit at index 0 for all plan cards. This may be a CMS/config change in the package picker content, or a frontend sort in the FeaturesGrid component.
CO-1.3 NOT IMPL Toggle button disabled on checkout page
View Screenshot
Root Cause: The checkout order summary currently has no collapse/expand toggle. The summary is always in expanded state showing full item details. The CDD requirement for collapsed (item count only) / expanded (details) states has not been implemented yet.
Location: Checkout summary component (FED code — likely in premium-domains or Eclipse checkout module)
Suggested Fix: Implement a toggle on the order summary component. Default state should be collapsed (show only item count, e.g. "2 items"). On click, expand to show full details. The cart state machine should track summaryExpanded: boolean.
CO-2.1 NOT IMPL Prices displayed in summary — CDD says show cycle only
View Screenshot
Root Cause: The checkout summary renders item prices ($9.90, $12.90, Total $22.80) in the expanded view. CDD requirement states expanded items should show billing cycle only (e.g. "1 year") without price.
Location: premium-store-core/.../PremiumPricesService.scala (BED) or checkout summary FED component
Suggested Fix: In the checkout summary expanded view, hide price fields and show only the billing cycle text. Either strip price fields from the API response for checkout context, or conditionally hide price in the FED component when rendering expanded summary items.
